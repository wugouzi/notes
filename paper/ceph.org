* System Overview
        The Ceph file system has three main components: the client, each instance of which exposes a
        near-POSIX file system interface to a host or process; a cluster of OSDs, which collectively stores
        all data and metadata; and a metadata server cluster, which manages the namespace (file names and
        directories) while coordinating security, consistency and coherence

        * *Decoupled Data and Metadata*. Metadata operations (open, rename, etc.) are collectively managed by a
          metadata server cluster, while clients interact directly with OSDs to perform file I/O (reads and
          writes).
